# 日報の書き方

## 日報の目的

日報は単なる報告ではありません。自身の業務の振り返りと、チームへの共有（ナレッジシェア）を目的としています。
困っていることや気づきを積極的に書きましょう。

## 推奨フォーマット

以下のような構成で書くことを推奨しています。

```markdown
# 今日の業務内容
- [x] プロジェクトAの資料作成
- [ ] クライアントBへのメール返信

# 所感・気づき
ここに今日の気づきや学びを書きます。

# 明日の予定
- クライアントBへのメール返信（午前中）
- チーム定例ミーティング
```

## 投稿手順

1. サイドバーの「日報」または「記事作成」をクリック。
2. テンプレートから「日報」を選択。
3. 内容を記入して「公開」をクリック。
